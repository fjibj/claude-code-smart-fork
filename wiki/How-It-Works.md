# 工作原理 (How It Works)

## 概述

Smart Fork 通过以下步骤实现智能会话搜索和切换：

1. **索引** - 将 Claude Code 会话保存到本地存储
2. **向量化** - 生成会话的语义向量表示
3. **搜索** - 使用向量相似度找到相关会话
4. **恢复** - 切换到你想要继续的会话

```

│                    /index-session                               │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 1: 读取 Claude Code 对话历史                                │
│                                                                 │
│  位置：~/.claude/projects/<project-name>/<session-id>.jsonl    │
│                                                                 │
│  • 解析 JSONL 格式的对话记录                                      │

└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 2: 生成会话摘要和关键词                                      │
│                                                                 │
│  • 如果用户提供 --summary，使用用户提供的摘要                      │
│  • 否则从对话内容自动生成摘要                                     │
│  • 提取关键主题 (keyTopics)                                      │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 3: 生成向量嵌入                                             │
│                                                                 │
│  • 将对话内容转换为文本                                          │

│  本地嵌入处理流程:                                                │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐     │

└─────────────────────────────────────────────────────────────────┘
                              │

│ Step 4: 保存到存储                                               │
│                                                                 │
│  • 会话数据：~/.smart-fork/sessions/<session-id>.json          │
│  • 索引更新：~/.smart-fork/index.json                          │

### 2. 语义搜索流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    /fork-detect                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 1: 提取当前上下文                                            │
│                                                                 │
│  • 当前工作目录                                                  │
│  • 当前 Git 分支                                                   │
│  • 最近打开的文件                                                │
│  • 用户提供的查询（如果有）                                       │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 2: 生成查询向量                                             │
│                                                                 │
│  • 将上下文信息组合成查询文本                                    │
│  • 使用相同的 EmbeddingService 生成查询向量                       │
│  • 确保查询和会话使用相同的向量空间                               │
└─────────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 3: 计算相似度                                               │
│                                                                 │
│  对于索引中的每个会话：                                           │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ cosine_similarity = (A · B) / (||A|| × ||B||)            │  │
│  │                                                          │  │
│  │ • A · B = 向量点积                                        │  │
│  │ • ||A|| = 向量 A 的 L2 范数                                  │  │
│  │ • ||B|| = 向量 B 的 L2 范数                                  │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
│  • 相似度范围：-1 到 1，越接近 1 越相似                             │
│  • 默认阈值：0.15（本地嵌入）                                    │
│  • 只保留超过阈值的会话                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 4: 排序并返回结果                                           │
│                                                                 │
│  • 按相似度分数降序排序                                          │
│  • 限制返回数量（默认 5 条）                                       │
│  • 显示结果并提供交互式选择                                      │
└─────────────────────────────────────────────────────────────────┘
```

│  • 验证会话完整性                                                │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 2: 保存当前上下文（用于 fork-back）                         │
│                                                                 │

                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 4: 显示会话历史                                             │
│                                                                 │
│  • 显示会话摘要                                                  │
│  • 打印对话历史（用户/助手交替）                                  │
│  • 显示工具调用记录（如有）                                      │
│  • 提供继续对话的上下文                                          │

```

## 中英文嵌入处理

### 本地嵌入器的多语言支持

```typescript
// 英文处理
英文文本 -> 分词（按空格） -> 过滤停用词 -> 哈希映射 -> 向量

// 中文处理
中文文本 -> 提取汉字 ->
  ├-> 单字切分 -> 哈希映射 -> 向量
  └-> 双字组合 (bigram) -> 哈希映射 -> 向量

// 混合文本
分别处理英文和中文部分，然后合并到同一个向量空间
```

### 停用词列表

**英文停用词：** the, a, an, is, are, was, were, have, has, had...

**中文停用词：** 的，了，是，在，我，有，和，就，不，人，都，一...

## 相似度计算示例

假设有两个会话：

**会话 A:** "npm install 403 forbidden 错误修复"
**会话 B:** "npm install 403 错误"
**会话 C:** "用户认证系统实现"

```
查询："npm install 403 错误"

计算余弦相似度：
- 会话 A: 0.45 (高度相关)
- 会话 B: 0.85 (非常相关)
- 会话 C: 0.02 (不相关)

如果阈值为 0.15，则返回会话 A 和 B
```

## 性能优化

1. **向量归一化** - 所有向量预先归一化，搜索时只需计算点积
2. **内存索引** - 启动时加载全部索引到内存
3. **批量嵌入** - 支持一次处理多个文本
4. **增量更新** - 只更新变化的会话，不影响其他数据
